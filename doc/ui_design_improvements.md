# UI 设计改进建议文档

## 1. 设计目标
基于 `example.jpeg` 参考设计，提升当前界面 (`shots.png`) 的视觉效果和用户体验，使其更加现代化、美观且一致。

## 2. 具体改进方案

### 2.1 边框圆角 (Border Radius)
- **现状**: 各元素圆角不统一
- **改进**: 统一使用以下圆角值
  - 大型容器: `12px` (如卡片、面板)
  - 中等元素: `8px` (如按钮、输入框)
  - 小型元素: `6px` (如标签、小图标)

### 2.2 卡片样式
- **背景**: 白色背景，保持简洁
- **边框**: `1px solid #E5E7EB` (浅灰色边框)
- **阴影**: 
  - 默认: `0 1px 3px rgba(0, 0, 0, 0.05), 0 1px 2px rgba(0, 0, 0, 0.03)`
  - 悬停: `0 4px 12px rgba(0, 0, 0, 0.08), 0 2px 4px rgba(0, 0, 0, 0.04)`
- **内边距**: 统一使用适当的内边距，保证内容呼吸空间

### 2.3 按钮设计
- **主要按钮 (Primary)**:
  - 背景: 渐变色 `linear-gradient(135deg, #F97316 0%, #FB923C 100%)`
  - 文字颜色: 白色
  - 边框: 无
  - 圆角: `8px`
  - 阴影: `0 1px 2px rgba(249, 115, 22, 0.2)`
  - 悬停效果: 更深的渐变 `linear-gradient(135deg, #EA580C 0%, #F97316 100%)`，阴影加深
- **次要按钮 (Secondary)**:
  - 背景: 白色
  - 边框: `1px solid #E5E7EB`
  - 文字颜色: 主文字颜色
  - 悬停效果: 背景变为 `#F9FAFB`，边框变为橙色 `#F97316`，文字变为橙色
- **幽灵按钮 (Ghost)**:
  - 背景: 透明
  - 文字颜色: 次要文字颜色
  - 悬停效果: 背景变为 `#F3F4F6`，文字颜色加深

### 2.4 颜色方案
- **主色调**: 橙色系列 (#F97316 为主要，#EA580C 为悬停)
- **辅助色**: 
  - 女性角色: 粉色系列 (#EC4899, #FDF2F8)
  - 男性角色: 蓝色系列 (#3B82F6, #EFF6FF)
- **背景色**: 浅灰 (#F9FAFB)
- **边框色**: 浅灰 (#E5E7EB)
- **文字色**: 深灰 (#111827) 和次级文字 (#6B7280)

### 2.5 排版与间距
- **字体**: Inter 或系统默认字体
- **行高**: 1.5 倍行高以提高可读性
- **字重**: 标题使用 500-600，正文使用 400
- **间距**: 使用一致的间距单位，如 4px、8px、12px、16px 等

### 2.6 交互反馈
- **悬停效果**: 所有可交互元素应有明显的悬停反馈
- **过渡动画**: 使用 `cubic-bezier(0.4, 0, 0.2, 1)` 作为标准缓动函数
- **选中状态**: 明确的选中/激活状态指示

### 2.7 组件特定样式
- **标签 (Tags)**: 圆角胶囊状，使用不同颜色区分类型
- **输入框**: 圆角边框，焦点状态有明确的边框颜色变化
- **表格/列表**: 行之间有适当间距，悬停状态有背景色变化
- **对话框/模态框**: 圆角边框，阴影效果，适当的内外边距

## 3. 实现优先级
1. **高优先级**: 边框圆角、卡片样式、主要按钮样式
2. **中优先级**: 颜色方案、排版间距
3. **低优先级**: 微交互动画、特殊组件样式

## 4. 注意事项
- 保持现有功能不变，仅优化视觉效果
- 确保所有修改都符合无障碍访问标准
- 在不同屏幕尺寸上测试响应式效果
- 保持性能，避免过度复杂的样式
