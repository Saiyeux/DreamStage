# AI短剧制作系统 - 设计文档

> 基于 ComfyUI + FLUX2 + LTX2 的AI短剧可视化制作流水线

---

## 一、系统概述

### 1.1 项目目标

构建一个可视化的AI短剧制作系统，实现从剧本上传到视频生成的完整工作流：

1. **剧本上传** → 解析PDF/TXT，生成剧情简介
2. **剧本分析** → 提取角色信息、分镜信息
3. **角色库生成** → 基于FLUX2生成角色参考图
4. **场景图生成** → 基于FLUX2生成分镜场景图
5. **视频生成** → 基于LTX2生成分镜视频片段
6. **视频合成** → 合成最终短剧视频

### 1.2 技术栈

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React + TypeScript + Tailwind | 现代化单页应用 |
| 后端 | FastAPI + Python 3.10+ | 异步API服务 |
| 本地LLM | Ollama / LM Studio | 剧本分析（OpenAI兼容API） |
| 图像生成 | ComfyUI + FLUX2 | 角色卡、场景图 |
| 视频生成 | ComfyUI + LTX-Video 2.0 | 分镜视频片段 |
| 数据存储 | SQLite + 本地文件系统 | 项目数据、生成资源 |

```python
from dataclasses import dataclass
from typing import List, Optional
from datetime import datetime
from enum import Enum

class ProjectStatus(Enum):
    DRAFT = "draft"
    ANALYZING = "analyzing"
    ANALYZED = "analyzed"
    GENERATING = "generating"
    COMPLETED = "completed"

@dataclass
class Project:
    id: str
    name: str
    script_path: str              # 原始剧本路径
    script_text: str              # 提取的文本内容
    summary: str                  # 剧情简介
    status: ProjectStatus
    created_at: datetime
    updated_at: datetime

@dataclass
class Character:
    id: str
    project_id: str
    name: str                     # 角色姓名
    gender: str                   # 性别
    age: str                      # 年龄
    role_type: str                # 主角/配角/龙套
    
    # 外貌特征 (用于生成Prompt)
    hair: str                     # 发型发色
    face: str                     # 脸型五官
    body: str                     # 身材体型
    skin: str                     # 肤色
    
    personality: str              # 性格特点
    clothing_style: str           # 服装风格
    scene_numbers: List[int]      # 出场场景编号列表
    
    # 生成的Prompt
    base_prompt: str              # 基础外貌Prompt
    
    # 生成的图像
    images: List['CharacterImage']
    main_image_id: Optional[str]  # 主参考图ID

@dataclass
class CharacterImage:
    id: str
    character_id: str
    image_type: str               # front/side/smile/surprised/thinking
    image_path: str               # 图像文件路径
    prompt_used: str              # 使用的Prompt
    negative_prompt: str          # 负面Prompt
    seed: int                     # 随机种子
    is_selected: bool             # 是否被选为参考图

@dataclass
class Scene:
    id: str
    project_id: str
    scene_number: int             # 场景编号
    
    # 场景基本信息
    location: str                 # 地点
    time_of_day: str              # 时间 (白天/黄昏/夜晚)
    atmosphere: str               # 氛围
    environment_desc: str         # 环境详细描述
    
    # 角色信息
    characters: List['SceneCharacter']

    shot_type: str                # 近景/中景/远景/特写
    camera_movement: str          # 镜头运动
    duration_seconds: int         # 预估时长(秒)
    
    # 生成的Prompt
    scene_prompt: str             # 场景图Prompt
    action_prompt: str            # 动作Prompt (视频用)
    negative_prompt: str          # 负面Prompt
    
    # 生成的资源
    scene_image: Optional['SceneImage']
    video_clip: Optional['VideoClip']

@dataclass
class SceneCharacter:

    position: str                 # 画面位置 (左/中/右)

class SceneImage:

## 五、核心Prompt设计

> ⚠️ **核心要点**: Prompt的质量直接决定生成效果，以下为精心设计的Prompt模板

### 5.1 剧情简介生成 Prompt

```markdown
你是一位专业的剧本分析师。请阅读以下剧本内容，生成一段简洁的剧情简介。

## 剧本内容
{script_text}

## 输出要求
请以JSON格式输出：
```json
{
  "summary": "100-200字的剧情简介，包含主要人物和核心冲突",
  "main_conflict": "主要冲突点（一句话）",
  "tone": "故事基调（如：甜宠、虐恋、悬疑、喜剧等）",
  "estimated_duration_minutes": 预估时长（分钟，数字）
}
```

要求：
1. 简介要引人入胜，突出戏剧冲突
2. 不要剧透结局
3. 语言简洁有力
```

### 5.2 角色分析 Prompt

```markdown
你是一位专业的剧本分析师和角色设计师。请从以下剧本中提取所有角色的详细信息。

## 剧本内容
{script_text}

## 输出要求
请以JSON格式输出所有角色：
```json
{
  "characters": [
    {
      "name": "角色姓名",
      "gender": "男/女",
      "age": "具体年龄或年龄段，如25岁、30多岁",
      "role_type": "主角/配角/龙套",
      
      "appearance": {
        "hair": "发型和发色，如：黑色长直发及腰、棕色短发微卷",
        "face": "脸型和五官特征，如：鹅蛋脸、杏眼、高鼻梁、薄唇",
        "body": "身材特征，如：身高170cm、身材修长、体型健壮",
        "skin": "肤色，如：白皙、小麦色、健康肤色"
      },
      
      "personality": "性格特点，用逗号分隔，如：温柔、独立、有主见",
      "clothing_style": "日常服装风格描述",
      
      "first_scene_number": 首次出场的场景编号（数字）,
      "total_scenes": 总出场场景数（数字）
    }
  ]
}
```

## 重要提示
1. **外貌描述要具体且适合AI图像生成**
   - 避免抽象描述如"美丽"、"帅气"
   - 使用具体特征如"高挺的鼻梁"、"深邃的双眼皮"
   
2. **如果剧本未明确描述外貌，请根据以下原则合理推断：**
   - 根据角色年龄、职业、性格推断外貌
   - 主角通常外貌出众

## 十、技术风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 角色一致性不足 | 不同场景角色外貌差异大 | 使用IPAdapter/PuLID，固定Seed，必要时训练LoRA |
| 视频生成质量不稳定 | 动作变形、画面崩坏 | 控制动作幅度，缩短时长，多次重试选优 |
| LLM分析不准确 | 角色/分镜信息错误 | 提供编辑功能，优化Prompt，使用更大模型 |
| 生成速度慢 | 用户体验差 | 队列管理，进度显示，后台异步生成 |
| 显存不足 | 无法加载模型 | 使用FP8量化模型，分批生成 |

## 附录: 推荐模型配置

### LLM (剧本分析)
- **Ollama**: qwen2.5:14b 或 qwen2.5:32b
- **LM Studio**: Qwen2.5-14B-Instruct 或 DeepSeek-V2.5

### 图像生成 (FLUX2)
- **模型**: FLUX.1-dev 或 FLUX.1-schnell
- **量化**: FP8 (节省显存)
- **显存需求**: 12GB+ (FP8), 24GB+ (FP16)

### 视频生成 (LTX2)
- **模型**: LTX-Video-2B
- **显存需求**: 16GB+
- **最佳帧数**: 97帧 (4秒@24fps)
