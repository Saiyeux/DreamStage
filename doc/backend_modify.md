# 后端服务优化
 - 现在后端服务有基本的工作能力，但远没有可以给用户使用的地步。

## 1. 分块显示
 - 角色和场景和剧幕，都应该分块显示。生成的时候是一次性生成所有内容，但显示的话，拿到一个json结果就显示一个。这样用户不必等完全解析。
 - 必须实时刷新，否则用户要主动刷新界面才能看到更新了哪些。要让用户感受到点一下分析就可以逐步获取的分析结果。

## 2. 分析逻辑优化
 - 现在角色/场景分析，往往是出现了角色名和地点后，llm开始根据上下文分析，然后给一个结果。问题在于，角色出场会有很多次，地点也会有很多次。所以最好是，用户上传一个文档之后，把文档给到模型，模型可以通过rag的方式，查询所以出场处，再具体分析。lm studio现在是有上传文件，rag的能力的。后端最后这样优化一下。

## 3. 剧幕分析
 - 现在还没实现剧幕分析，需求如下：
 - 根据关键词，先检测一共有多少幕，区分每一幕的文本
 - 然后对每一幕，进行分析，总结一个大致标题和简介，显示在act analysis那里
 - 在script ref那里放原文
 - 这样设计的原因是，act为用户真正作为导演开拍的地方，不用分析过多，只需要让导演编辑，然后生成视频就行。
 - 剧本里有很多关键词，如chapter, scene, act等，把这个作为前端可配置内容，用户上传完整剧本的时候，配置具体检测词。让llm智能分幕。
